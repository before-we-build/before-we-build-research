# Аудит современных методик типирования для агентного пайплайна публичных фигур

Статус: **методологический аудит**, не список окончательно валидированных инструментов. Цель — проверить, использует ли [[public-figure-typing-agent-pipeline-ru]] самые перспективные доступные подходы и не переоценивает ли интернет-тесты, LLM-анализ текста или типировочные сообщества.

## Короткий вывод

Пайплайн в целом соответствует лучшей доступной стратегии для публичных фигур:

```text
много источников → наблюдаемые признаки → независимые гипотезы по системам → критика role-overfit → арбитраж уверенности
```

Но его нужно дополнить явной матрицей методик:

- **соционика:** интервью, Model A / информационные аспекты, дихотомии, письменные тесты, Talanov inventory, внешняя критика;
- **психософия:** три оси/дихотомии, интервью, Afanasyev/Anette/AP-подобные опросники как слабые источники, анализ уязвимости/процессности/результативности;
- **темпористика:** aspect × position coding, открытые нарративы о прошлом/настоящем/будущем/вечности, сценарные вопросы; из-за меньшей публичной методической базы уверенность должна быть ниже;
- **AI/text analysis:** использовать только как вспомогательный извлекатель признаков и проверку повторяющихся языковых паттернов, не как прямой типировщик;
- **психометрика:** любые тесты считать не “истиной”, а одним источником, пока нет reliability/validity, межэкспертной согласованности и внешней проверки.

## Проверенные внешние источники и что из них следует

### Wikisocion: type diagnosis

Источник `https://wikisocion.github.io/content/diagnosis.html` прямо предупреждает, что описания типов лучше рассматривать как дидактический инструмент, а не как инструмент типирования. Там также отмечается проблема письменных тестов: они пытаются объективировать типирование через дихотомии, но трудно передать тестируемому смысл дихотомий и элементов так, чтобы ответы действительно мерили тип, а не понимание слов.

Вывод для пайплайна:

- тесты и описания типов не должны быть главным доказательством;
- нужен interview / evidence coding / external critique;
- важно отличать “узнал себя в описании” от типировочного признака.

### Sociotype.com tests

Источник `https://sociotype.com/tests/` показывает живую интернет-практику тестов и типовых профилей. Это полезно как карта популярных онлайн-инструментов, но не как валидирующий стандарт.

Вывод:

- можно использовать тестовые шкалы как подсказку для вопросов;
- нельзя переносить тестовый результат публичной фигуры без самопрохождения и проверки;
- для биографического типирования это только слабая методическая аналогия.

### AIM / shortened Talanov inventory

Источник `https://www.aimtoknow.com/test_beta` описывает shortened online form of V. Talanov inventory for socionics type detection.

Вывод:

- Talanov-style multi-scale inventory перспективнее простого “угадай тип по описанию”, потому что работает как набор шкал;
- но для публичных фигур, которые не проходят тест, он полезен только как модель того, какие признаки искать;
- пайплайн должен добавить “шкальный слой” для соционики: не только тип, но сигналы аспектов/дихотомий/Model A.

### Attitudinal Psyche / Psychosophy-adjacent theory

Источник `https://www.attitudinalpsyche.com/theory/` даёт современную англоязычную ветвь, близкую к психософии: аспекты, позиции/attitudes, desire/fear, confident/insecure, process/result и т.п.

Вывод:

- полезно как параллельная современная терминологическая проверка;
- нельзя смешивать AP и Афанасьева как один источник;
- для пайплайна полезны оси: уверенность/уязвимость, процесс/результат, самостоятельность/адаптация, желание/страх.

### Психософские источники repo

`psychosophy-typing-methods.md` уже фиксирует:

- тест Афанасьева;
- большой опросник Anette;
- типировочную анкету;
- три дихотомии: верх/низ, процесс/результат, доминирование/подстройка;
- необходимость внешней проверки несколькими типировщиками;
- отсутствие установленной psychometric validity для community-опросников.

Вывод:

- наш пайплайн должен явно требовать кодирования по трём осям, а не только угадывания четырёхбуквенного типа;
- для публичных фигур нужно искать следы 1/2/3/4 позиции в поведении: стандарт, процессная помощь, болезненная проверка, делегируемая рамка.

### Темпористика

Публичная методическая база по темпористике заметно слабее, чем по соционике и психософии. В repo есть `temporistics-test-specification.md`, где уже предложена перспективная структура:

- aspect salience: Past / Present / Future / Eternity;
- position modes: Target / Creative / Painful / Blind;
- aspect × position heatmap;
- сценарии: переход, потеря смысла, релокация, конфликт, слом маршрута.

Вывод:

- темпористические типы публичных фигур должны иметь более низкую уверенность по умолчанию;
- нужно требовать не только “говорит о будущем/вечности”, а именно позиционный режим: сам задаёт, процессно развивает, болезненно проверяет или берёт как ресурс;
- у христианских служителей Вечность может быть жанром веры и богословия, а не 1Вечность.

### AI / LLM / text-based automatic personality prediction

Проверенные arXiv-источники показывают перспективность, но также ограничения:

- `2310.10679` — LLM может реплицировать cross-cultural Big Five differences, но есть upward bias, lower variation и lower structural validity.
- `2406.17287` — LLM может предсказывать Big Five из counselling dialogues с заметной корреляцией, особенно через role-play/questionnaire prompting.
- `2403.07581` — LLM полезен как text augmentation / analysis для personality detection, даже когда сам LLM не является хорошим прямым классификатором.
- `2605.17355` — text-based automatic personality prediction развивается через multi-level текстовые модели: документ, предложение, слово.
- `2110.01186` — APP давно развивается в NLP, включая pretrained/deep learning/multimodal approaches, но это область prediction/perception, не прямое доказательство типологии.

Вывод:

- LLM полезен как **извлекатель признаков**, кодировщик повторяющихся тем и генератор альтернатив;
- LLM не должен быть “типировщиком последней инстанции”;
- для публичных фигур нужен контроль жанра, авторства текста, редакторской обработки, перевода, даты, аудитории;
- если использовать LLM-анализ текста, нужно хранить цитаты/фрагменты, по которым сделан вывод.

## Матрица методик для пайплайна

| Методика | Где полезна | Сила | Риск | Как использовать |
|---|---|---:|---|---|
| Официальная биография | source dossier, роль, траектория | medium для фактов, weak для типа | role-overfit | только как контекст и слабые следы |
| Полное интервью | все 3 системы | strong | самопрезентация, жанр | кодировать выбор, конфликт, смысл, временные рамки |
| Полная проповедь/лекция | соционика, темпористика, психософия | medium-strong | церковный жанр, тема текста | сравнивать несколько тем и лет |
| Книги/статьи | соционика, темпористика | medium | редактура, жанр, тема | искать устойчивую структуру мышления, не только тему |
| Соционическое интервью | соционика | strong при хорошем интервьюере | bias типировщика | отдельный агент + критик |
| Соционические дихотомии / Model A | соционика | medium-strong | натягивание признаков | кодировать аспекты и функции отдельно до типа |
| Письменные соционические тесты | соционика | weak-medium | понимание терминов, self-typing bias | только как дополнительный self-report, если человек сам прошёл |
| Talanov-style шкалы | соционика | medium как методическая модель | нельзя применить к биографии напрямую | использовать как список шкал/признаков |
| Afanasyev / Anette / AP-подобные опросники | психософия | weak-medium | невалидированность, self-report | только при самопрохождении + интервью |
| Три оси психософии | психософия | medium-strong | путаница роли и позиции | кодировать по каждому аспекту до типа |
| Aspect × position heatmap | темпористика | medium | мало внешней методической базы | обязательна для темпористики |
| LLM-анализ текста | все системы как вспомогательный слой | medium для извлечения признаков, weak для типа | hallucination, genre bias, авторство/редактура | только trace extraction + alternatives, не финальный тип |
| Независимые типировщики | все системы | medium-strong | shared bias, groupthink | разделять агентов и затем критиковать |
| Role-overfit critic | все системы | strong | может чрезмерно занизить уверенность | обязателен для лидеров/пасторов/богословов |

## Требуемое обновление пайплайна

[[public-figure-typing-agent-pipeline-ru]] должен явно включать следующие слои:

1. **Method scout** — перед типированием проверяет, какие методики релевантны конкретной системе.
2. **Scale coding before type coding** — сначала признаки/шкалы/оси, потом тип.
3. **Text-analysis auxiliary layer** — LLM извлекает темы и языковые паттерны из текстов, но не решает тип.
4. **Role/genre/source-quality weights** — каждый источник получает вес: official bio, sermon, interview, book, article, edited text, social media.
5. **System-specific warning rules:**
   - соционика: не выводить тип из MBTI-like ярлыков или описаний;
   - психософия: не выводить 1В из должности и 1Л из образования;
   - темпористика: не выводить 1Вечность из религиозной лексики.
6. **Confidence gate:** если есть только биография и проповеди одного жанра, максимум `низкая-средняя`; если только официальная роль, максимум `низкая`.

## Практический вывод для текущих баптистских типирований

- Для Валерия Антонюка и других союзных руководителей обязательно нужен role-overfit downgrade: должность имитирует ЛСЭ/ВЛФЭ/ПБНВ.
- Для Сергея Санникова можно использовать books/articles как сильный материал по структуре мышления, но всё равно нужен живой material для конфликта/выбора.
- Для Николая Романюка нужно сравнить проповеди, пасторские интервью и лидерские материалы, чтобы отделить ЭИИ от ЛИИ.
- Для Игоря Бандуры нужно брать материалы вне темы духовного отцовства, иначе pipeline overfits на одну конференционную тему.
- Для темпористики всех служителей уверенность должна быть ниже, пока нет явных данных по позиции Вечности/Будущего/Прошлого за пределами обычного христианского языка.
