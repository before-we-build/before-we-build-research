# Протокол доказательного типирования публичных фигур

Статус: **рабочий исследовательский протокол**. Он нужен, чтобы следующий этап типирования публичных баптистских / евангельских фигур был точнее, чем общий биографический проход. Протокол не делает типирование окончательным и не превращает типологию в духовную, моральную, пасторскую или кадровую оценку.

Базовая формула проекта сохраняется: **типовые характеры, не типы человека**.

## Зачем нужен протокол

Текущие типирования уже прошли через агентный пайплайн и методологический аудит, но большая часть источников всё ещё слабая: официальные биографии, представительские страницы, тематические материалы. Они показывают служебный путь, но плохо показывают:

- живой способ мышления;
- реакцию на несогласие;
- стиль выбора в неопределённости;
- способ помощи людям;
- личный temporal mode: прошлое, настоящее, будущее, вечность;
- отличие личного паттерна от должности, жанра проповеди и церковной культуры.

Следующий скачок точности требует не ещё одного общего пересказа, а **цитатной базы + независимого кодирования + арбитража уверенности**.

## Минимальный corpus на одну фигуру

### Слабый, но допустимый pass

- 1 официальная биография;
- 1 полная проповедь, лекция, интервью или текст, где человек рассуждает не только формально по должности;
- явно отмеченный role-overfit.

Максимум уверенности: `низкая` или `низкая-средняя`.

### Хороший исследовательский pass

- официальная биография;
- 3–5 полных проповедей / лекций / интервью разных лет;
- минимум 2 жанра: например проповедь + интервью, лекция + Q&A, книга + публичная дискуссия;
- 1 материал в кризисном, конфликтном или decision-making контексте;
- отдельный critic pass по role-overfit;
- независимое второе кодирование хотя бы части evidence rows.

Максимум уверенности: `средняя`, но только если альтернативы реально ослаблены.

### Сильный, но редко доступный pass

- живое интервью или анкета;
- self-report по дифференциаторам;
- observer report от людей, знающих фигуру лично;
- независимые типировщики, не видящие выводы друг друга;
- проверка по материалам разных лет.

Даже здесь для публичных служителей лучше избегать слова `высокая`, если нет согласия самого человека и независимой проверки.

## Вес источников

| Источник | Вес для фактов | Вес для типа | Ограничение |
|---|---:|---:|---|
| Официальная биография | 3 | 1 | показывает роль и траекторию, а не живой стиль |
| Представительская страница | 3 | 1 | часто PR/институциональный жанр |
| Полная проповедь | 2 | 2 | жанр Писания/проповеди может имитировать Вечность, Эмоцию, Логику |
| Серия проповедей разных тем | 2 | 3 | лучше показывает устойчивость паттерна |
| Интервью | 2 | 3 | показывает выбор, реакцию, самоописание; но есть самопрезентация |
| Q&A / дискуссия | 2 | 4 | сильнее показывает спонтанную обработку несогласия |
| Книга / статья | 2 | 2–3 | хорошо для структуры мысли; слабо для интеракции |
| Кризисное обращение | 2 | 3 | показывает приоритеты под давлением; но кризис искажает стиль |
| Соцсети | 1 | 1–2 | короткий формат, редактура, публичный образ |
| Внешнее типирование | 1 | 1–2 | полезно только с аргументацией и источниками |
| Интернет-тест | 1 | 1–2 | только если человек сам проходил; без psychometric validation не доказательство |

Каждая evidence row должна хранить не только вывод, но и фрагмент источника.

## Blind coding procedure

Чтобы уменьшить bias от известности, должности и симпатии, часть материалов нужно кодировать слепо.

### Blind packet

Для blind packet удалить или скрыть:

- имя;
- должность;
- название церкви / союза, если оно раскрывает роль;
- прямые биографические подсказки;
- уже предложенный тип.

Оставить:

- фрагмент речи;
- жанр в общем виде: проповедь, интервью, лекция, Q&A;
- год или период, если он важен для кризисного контекста;
- минимальный контекст темы.

### Blind coding steps

1. Coder A кодирует только observable traces.
2. Coder B независимо кодирует те же фрагменты.
3. Arbiter сравнивает совпадения и расхождения.
4. Только после этого возвращается имя/должность.
5. Role-overfit critic проверяет, что изменилось после раскрытия контекста.

## Inter-rater agreement

Для каждого спорного вывода фиксировать:

```yaml
agreement:
  coder_a: "ЛИИ > ИЛИ"
  coder_b: "ИЛИ > ЛИИ"
  coder_c: "ЛИИ, но weak"
  agreement_level: low | partial | strong
  arbitration: "оставить primary ЛИИ, но confidence не выше низкая-средняя"
```

Правило:

- strong agreement + diverse sources → можно повышать уверенность;
- partial agreement → сохранять альтернативы;
- low agreement → не выбирать тип силой, а писать `неразличимо по текущим данным`.

## Confidence model

Оценка не должна быть процентом. Она даёт класс уверенности.

| Фактор | Баллы |
|---|---:|
| Source breadth: 1 источник / 2–3 / 4+ | 0–3 |
| Genre diversity: один жанр / два / три+ | 0–3 |
| Spontaneity: подготовленный текст / интервью / Q&A-дискуссия | 0–3 |
| Cross-year stability: один период / 2 периода / 3+ лет | 0–3 |
| Crisis or conflict material: нет / косвенно / явно | 0–2 |
| Evidence specificity: общие роли / фрагменты / цитаты с кодировкой | 0–3 |
| Independent agreement: нет / partial / strong | 0–3 |
| Role-overfit risk: low / medium / high | 0 / -1 / -3 |
| Genre contamination risk: low / medium / high | 0 / -1 / -2 |
| System contamination risk: low / medium / high | 0 / -1 / -2 |

Классы:

| Итог | Класс |
|---:|---|
| 0–3 | очень низкая |
| 4–6 | низкая |
| 7–9 | низкая-средняя |
| 10–12 | средняя |
| 13+ | средняя с пометкой `требует живой проверки`; не писать `высокая` без интервью/self-report |

## Differentiator checklists

### ЛСЭ vs ЛСИ vs ЭСИ

Проверять:

- `ЛСЭ`: говорит через рабочие процессы, практическую полезность, организацию, результат, устойчивый быт/ритм;
- `ЛСИ`: говорит через порядок, норму, структуру, дисциплину, границу системы, подчинение принципу;
- `ЭСИ`: говорит через верность, доверие, личные границы, защиту своих, этическую ответственность.

Красный флаг: председательская или административная должность может имитировать все три.

### ЭИИ vs ЛИИ

Проверять:

- `ЭИИ`: ведёт через совесть, личную зрелость, доверие, отношения, внутреннюю честность, пасторскую чувствительность;
- `ЛИИ`: ведёт через категории, структуру учения, системное объяснение, внутреннюю непротиворечивость.

Красный флаг: пастор-учитель может выглядеть и ЭИИ, и ЛИИ.

### ЭИЭ vs ЭСЭ

Проверять:

- `ЭИЭ`: поколение, миссия, драматургия будущего, вдохновение через смысл и напряжение пути;
- `ЭСЭ`: атмосфера общины, тепло, включение людей, практическая эмоциональная забота здесь и сейчас.

Красный флаг: конференционная тема духовного отцовства может искусственно усиливать ЭИЭ.

### СЭЭ vs СЛЭ vs ЭИЭ

Проверять:

- `СЭЭ`: личное влияние, защита конкретных людей, отношение + действие;
- `СЛЭ`: силовая организация ситуации, давление, управление ресурсом и риском;
- `ЭИЭ`: эмоционально-смысловая мобилизация вокруг миссии.

Красный флаг: капелланство, социальная работа и кризис могут усиливать Se/Future независимо от типа.

### ЛВЭФ vs ЛЭВФ

Проверять:

- `ЛВЭФ`: первична собственная логическая рамка, 2В строит институты и проекты;
- `ЛЭВФ`: первична логика, но второй процесс — эмоционально-педагогическое ведение аудитории.

Красный флаг: преподавательская роль может имитировать обе.

### ВЛФЭ vs ЛВФЭ

Проверять:

- `ВЛФЭ`: первично удержание направления/ответственности; логика служит процессной проработке;
- `ЛВФЭ`: первичен принцип/модель/порядок; воля процессно поддерживает реализацию.

Красный флаг: руководящая должность может имитировать 1В.

### ВФЭЛ vs ФВЭЛ

Проверять:

- `ВФЭЛ`: запуск и удержание дела через волевую рамку, практический слой служит ей;
- `ФВЭЛ`: первичен стандарт земной устойчивости, быта, ресурса, телесно-материальной организации.

Красный флаг: ремесло или практичность не доказывают 1Ф.

### ПБНВ vs БПНВ

Проверять:

- `ПБНВ`: идентичность/преемственность задаёт рамку; будущее разворачивается из прошлого;
- `БПНВ`: будущая миссия задаёт рамку; прошлое служит материалом и ресурсом.

Красный флаг: церковные организации часто говорят о преемственности и миссии одновременно.

### ВПБН vs ПВБН

Проверять:

- `ВПБН`: смысл/вечность задаёт рамку, история помогает её раскрывать;
- `ПВБН`: история/зрелость/преемственность задаёт рамку, вечность помогает её осмыслить.

Красный флаг: христианская лексика не доказывает 1Вечность.

### БНПВ vs НБПВ

Проверять:

- `БНПВ`: будущая миссия задаёт направление, настоящее — поле действия;
- `НБПВ`: текущее место/община/среда задаёт рамку, будущее раскрывается из неё.

Красный флаг: church planting может переобучать на 1Будущее.

## Итоговый отчёт по фигуре

```markdown
## Имя Фамилия — evidence dossier

### Source set
- список источников с весом

### Evidence table
- 10–30 rows с фрагментами и кодировкой

### Blind coding result
- что видно без имени/должности

### System summaries
- Socionics: scale/aspect profile before type
- Psychosophy: axes before type
- Temporistics: aspect × position heatmap before type

### Differentiator result
- какие спорные пары ослаблены/усилены

### Confidence model
- баллы и класс

### Arbitration
- primary, alternatives, confidence, what would change the conclusion
```

## Приоритет применения

Начать с трёх фигур, где потенциально больше материалов:

1. **Сергей Санников** — книги, сайт, лекции, богословские тексты; хорош для проверки ЛИИ/ИЛИ и ЛВЭФ/ЛЭВФ.
2. **Николай Романюк** — церковный сайт и вероятный корпус проповедей; хорош для проверки ЭИИ/ЛИИ.
3. **Игорь Бандура** — уже есть сильная тема духовного отцовства; нужно проверить её вне одного жанра.

Только после этого расширять на остальных лидеров ВСЦ ЕХБ, потому что по ним пока преобладают официальные биографии.
